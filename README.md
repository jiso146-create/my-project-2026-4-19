SilentGuardian-Redundancy-Protocol (SGRP)室内活動センサーと物理鍵Beaconの相関による多重判定安否監視ロジック（v2.0）本リポジトリは、独居居住者の孤立死およびそれに伴う不動産の資産価値毀損リスクを最小化するための、高精度な安否監視アルゴリズム「SGRP」の仕様を公開するものです。

1. 開発の背景と目的既存の見守りサービスは「救命」を主目的とするあまり、不動産管理実務における「経済的損失の回避」という視点が欠落しています。
2. 固定閾値の限界: 一般的な「24時間ルール」では、夏季の遺体腐敗（キャスパーの法則）による損害を防げない。
3. 在宅・外出判定の不備: 居住者の自己申告や低精度な判定により、外出中の無活動を「異常」と誤認する誤報が多発している。
4. 本プロジェクトは、**「室内での生活動態」と「物理鍵（Beacon）の有無」**を多重に突合することで、この問題を解決します。

技術の特徴（コアロジック）
A. 室内活動センサーとBeaconの相関判定室内（リビング、トイレ等）に設置した「活動検知センサー（人感・開閉等）」と、居住者が外出時に必ず携行する「物理鍵（Beacon）」の状態を組み合わせ、以下の論理階層で安否を判定します。
活動検知（Primary Indicator）: 室内センサーが何らかの活動を検知している限り、安否は「正常」とみなす。
無活動タイムアウト: 室内での活動が一定時間（閾値 $T$）途絶えた場合、次にBeaconの状態を参照する。Beaconによる文脈判定（Redundancy Check）:Beacon＝在宅: 「家の中にいるのに、一定時間動きがない」→ 【異常事態：アラート発報】Beacon＝消失: 通常は「外出」とみなすが、**「扉の開閉がないままBeacon信号のみが消失した」**場合は、屋内での転倒による遮蔽や機器故障の可能性があるため、安否確認の重要度を維持する。

B. 環境適応型・動的無活動閾値（Adaptive Threshold）無活動を「異常」と断定するまでの閾値 $T$ を、法医学的リスクに基づき動的に変更します。夏季（6月〜9月）: 腐敗による汚損回避を優先し、$T = 12時間$ に設定。
冬季（それ以外）: 誤報抑制を考慮し、$T = 24時間$ に設定。※管理者の設定により、居住者のリスク特性に応じた任意の閾値変更を可能とします。

C. 二重プロトコルによる通信冗長化RC造マンション等の高度な遮蔽環境におけるBeacon信号のロスト（偽の外出判定）を防ぐため、以下の二重化プロトコルを併用し、在席判定の信頼性を高めます。
標準BLE (Bluetooth Low Energy)低レイヤー独自通信 (ESP-NOW等)

        [最終活動時刻] = NOW
        [アラートフラグ] = FALSE
        CONTINUE

    // 無活動時間の算出
    [無活動時間] = NOW - [最終活動時刻]

    IF [無活動時間] > T (動的閾値) THEN
        // 在宅判定の精査
        IF [Beacon受信] == TRUE THEN

                [アラート通知] (閉じ込め・屋内転倒による遮蔽の疑い：要確認)
            ELSE
                [監視ステータス] = PAUSE (正常な外出と判定)
            END IF
        END IF
    END IF

    // 季節による閾値の動的変更
    IF [CurrentMonth] IN [6, 7, 8, 9] THEN T = 12h ELSE T = 24h
   
6. 自己修復・死活監視機構ゲートウェイ（受信機）は、内部に独立したウォッチドッグタイマー（WDT）を維持します。Wi-Fiのスタックやメモリリーク等で通信が一定時間途絶えた場合、自律的にハードウェアリセットを実行し、24時間365日の継続稼働を担保します。
