SilentGuardian-Redundancy-Protocol (SGRP)室内活動と物理鍵の相関判定、および任意閾値制御を備えた動的安否監視システム（v2.9）

1. 開発の背景：資産価値毀損の防止と社会課題の解決本システムは、独居世帯における孤立死の早期発見、および発見の遅れに伴う不動産資産価値の毀損（事故物件化）を防止するために開発された。孤立死による経済的実害: 発見の遅れは遺体の腐敗を招き、多額の特殊清掃費用、家賃減額、告知義務による資産価値の長期的損失を発生させる。12時間のデッドライン: 法医学的知見（キャスパーの法則）に鑑み、夏季の密閉空間では24時間以上の放置は致命的な汚損を招く。本システムは「12時間単位」での確実な検知を目的とする。運用負荷の軽減: 既存の監視システムにおける「外出」と「異常」の誤認を排除し、管理会社等の空振り訪問によるコストを削減する。

. システム物理構成（Hardware Architecture）
エッジ・ゲートウェイ:
ESP32-C6等のマイコンを核とした常時稼働ノード。Wi-Fi 6 / Bluetooth 5 (LE) / 802.15.4 (Zigbee/Thread) 等のマルチプロトコルに対応し、WDT（ウォッチドッグタイマー）によるハングアップ時の自己修復機構を具備。

室内動態センサー（プライマリ）:
人感（PIR）、電力（クランプ式電流センサ）、照度等を検知。居住者の生存を証明する一次指標として機能。

多機能識別ビーコン（インテリジェント・タグ）:
物理鍵等に装着されるBLE送信機。信号の有無による在宅判定に加え、必要に応じて加速度センサーを内蔵。信号消失直前の「静止・移動」の物理状態をログとして保持し、外出か遮蔽かを判別する二次指標を提供する。

補助センサー（オプション）:
玄関扉の開閉を検知するリードスイッチ、または共用部に設置されたAIカメラ、オートロック解錠ログ、エマージェンシー・システム等の外部インフラ。これらを「外出・帰宅のイベントトリガー」として判定ロジックに組み込むことで、住戸ごとのハードウェアコストを抑制しつつ精度を補完する。

3. 通信プロトコル（Communication Protocols）BLE (Bluetooth Low Energy): 標準的なアドバタイジング信号のスキャンにより、ビーコンの近接および存在確認を行う。ESP-NOW: Wi-Fiルーター等の外部インフラに依存しない低レイヤーの独自通信プロトコル。壁などの遮蔽物によるパケットロスを最小化し、ゲートウェイ間やセンサー間の通信冗長性を確保する。4. 安否判定ロジック（SGRP Algorithm）室内活動（主指標）とビーコン状態（従指標）を相関させ、物理的な矛盾を抽出する多重判定構造を採用。物理相関による矛盾検知:室内無活動が検知された際、単に「信号の有無」で判断せず、Beacon内蔵の加速度センサーによる移動履歴や、玄関扉の開閉履歴と突合。これにより、「置いたままの鍵（在宅）」と「持ち出された鍵（外出）」を物理的に峻別し、誤報を排除する。閾値 $T$ の任意性（Dynamic Threshold）:無活動を異常と見なす時間 $T$ は、居住者のライフスタイルや法医学的リスク（季節変動）に基づき、運用者が任意の値にいつでも設定・変更可能。夜間通知保留（Notification Scheduling）:深夜帯（例 22:00〜06:00）に発生したアラートをシステム内で蓄積。管理者のQOL（生活の質）と実務負荷を考慮し、緊急時を除き翌朝の指定時刻に一括通知する。
   
【判定疑似コード】PlaintextLOOP:
  LOOP:
    T = GetSetting(RoomID) // 運用者設定の無活動閾値（例：12時間）
    K = 0.5 // 加速度しきい値（移動の有無を判定）

    // --- 1. 生存確認（プライマリ） ---
    IF [室内センサー活動検知] == TRUE THEN
        [最終活動時刻] = NOW
        [アラートフラグ] = FALSE
        CONTINUE

    [無活動時間] = NOW - [最終活動時刻]

    // --- 2. 異常判定フェーズ ---
    IF [無活動時間] > T THEN
        IF [Beacon受信] == TRUE THEN
            // 在宅しているが、一定時間動きがない
            [アラート内容] = "在宅中・無活動（緊急：室内での異変）"
            [アラートフラグ] = TRUE
        ELSE
            // Beacon信号が消失：その理由を物理相関で検証
            // ※外出トリガー ＝ ドア開閉、共用部カメラ通過、オートロック解錠など
            
            IF [ΔA_max] < K AND [外出トリガー] == FALSE THEN
                // 鍵が動かず、外出の形跡もないのに電波が消えた ＝ 遮蔽・屋内転倒
                [アラート内容] = "信号消失矛盾：屋内遮蔽・停滞の疑い（要確認）"
                [アラートフラグ] = TRUE
            
            ELSE IF [ΔA_max] > K AND [外出トリガー] == TRUE THEN
                // 鍵が動き、外出の形跡を伴って電波が消えた ＝ 正常な外出
                [監視ステータス] = PAUSE
                [アラートフラグ] = FALSE
            
            ELSE
                // どちらとも言えない不確定な消失（例：鍵を忘れて外出した、電波の瞬間的な死角など）
                [アラート内容] = "安否不明：信号消失プロセス異常（要確認）"
                [アラートフラグ] = TRUE
            END IF
        END IF
    END IF

    // --- 3. 通知制御（夜間保留プロトコル） ---
    IF [アラートフラグ] == TRUE THEN
        IF [現在時刻] NOT IN [夜間保留時間帯] THEN
            [通知実行] 
        ELSE
            [キュー保存] // 翌朝の指定時刻に一括送信
        END IF
    END IF
    
4. 知的財産権保護（防御的開示宣言）以下の技術的思想および構成は、2026年4月25日時点において公知の事実（先行技術）となる。本開示の目的は、独居者の安否監視における標準的な技術基盤を確立し、特定の企業による不当な技術独占を防止することにある。物理的移動ベクトルと信号消失の相関判定:室内活動の停止に対し、物理鍵（Beacon）の信号消失を単一のフラグとせず、消失直前の加速度変化量 $\Delta A_{max}$ としきい値 $K$ を突合。さらに玄関ドアセンサーの開閉履歴を相関させ、「物理的な移動（ドア開・加速度大）」を伴わない信号消失を「外出」ではなく「屋内での異常（遮蔽・転倒・故障）」と定義する多重矛盾検知ロジック。法医学的知見に基づく動的閾値管理:死後腐敗の進行速度（キャスパーの法則）および外気温等の環境要因を考慮し、安否判定の無活動閾値 $T$ を運用者が任意かつ動的に変更可能とする機構。実務最適化通知プロトコル:異常検知時の通知を、管理者の実務負荷軽減のために特定の時間帯（夜間等）において保留・蓄積し、指定時刻に一括送信する運用管理用プロトコル。通信経路の二重化と低レイヤー制御:標準BLE（Bluetooth Low Energy）によるスキャニングと、低レイヤー独自通信（ESP-NOW等）を併用し、ネットワーク障害や遮蔽環境下においても安否情報の到達性を確保する冗長化プロトコル。エッジ側での自律判定機構:クラウドへの依存を最小限に抑え、エッジゲートウェイ（ESP32-C6等）側で上記の矛盾検知アルゴリズムを完結させることにより、通信遅延やサーバーダウンの影響を受けない自律的な監視インフラの構成。
共用部インフラとの動的連携プロトコル:住戸ごとの個別センサーに加え、建物共用部に設置された監視カメラの動体検知、顔認証、またはスマートロックの操作ログを「外出の確証（トリガー）」として受信し、住戸内のBeacon信号の消失と相関判定を行うことで、システム全体の導入コストを最適化しつつ判定精度を担保する構成。
