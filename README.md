SilentGuardian-Redundancy-Protocol (SGRP)室内活動と物理鍵の相関判定、および任意閾値制御を備えた動的安否監視システム（v2.7）

1. 開発の背景：資産価値毀損の防止と社会課題の解決本システムは、独居世帯における孤立死の早期発見、および発見の遅れに伴う不動産資産価値の毀損（事故物件化）を防止するために開発された。孤立死による経済的実害: 発見の遅れは遺体の腐敗を招き、多額の特殊清掃費用、家賃減額、告知義務による資産価値の長期的損失を発生させる。12時間のデッドライン: 法医学的知見（キャスパーの法則）に鑑み、夏季の密閉空間では24時間以上の放置は致命的な汚損を招く。本システムは「12時間単位」での確実な検知を目的とする。運用負荷の軽減: 既存の監視システムにおける「外出」と「異常」の誤認を排除し、管理会社等の空振り訪問によるコストを削減する。

2. システム物理構成（Hardware Architecture）エッジ・ゲートウェイ: ESP32-C6等のマイコンを核とした常時稼働ノード。WDT（ウォッチドッグタイマー）による自己修復機構を具備。室内動態センサー: 人感、電力、照度等を検知する生存確認の主指標（プライマリ・センサー）。識別ビーコン: 物理鍵等に装着される送信機。在宅・外出のコンテキストを提供する。補助センサー（オプション）: 玄関扉等の開閉を検知するリードスイッチ。判定精度の補完に用いる。

3. 通信プロトコルBLE (Bluetooth Low Energy): 標準信号スキャン。ESP-NOW: 低層独自通信による遮蔽環境下での通信冗長化。4. 安否判定ロジック（SGRP Algorithm）室内活動（主）とビーコン（従）を組み合わせ、扉開閉履歴等の矛盾を突合する多重判定構造。閾値 $T$ の任意性: 無活動を異常と見なす時間 $T$ は、運用者が任意の値にいつでも設定・変更可能である。夜間通知保留: 特定の時間帯（例：22:00〜06:00）のアラートを蓄積し、翌朝に一括通知する。

【判定疑似コード】PlaintextLOOP:
    // 運用者設定の閾値Tを取得
    T = GetSetting(RoomID) 

    IF [室内センサー活動検知] == TRUE THEN
        [最終活動時刻] = NOW
        [アラートフラグ] = FALSE
        CONTINUE

    [無活動時間] = NOW - [最終活動時刻]

    // 異常判定
    IF [無活動時間] > T THEN
        IF [Beacon受信] == TRUE THEN
            [アラート内容] = "在宅中・無活動（緊急）"
            [アラートフラグ] = TRUE
        ELSE
            // 矛盾検知（扉履歴参照）
            IF [ドアセンサーあり] AND [最終ドア開閉] == "閉" THEN
                [アラート内容] = "屋内転倒・遮蔽の疑い（要確認）"
                [アラートフラグ] = TRUE
            ELSE
                [監視ステータス] = PAUSE (正常な外出)
            END IF
        END IF
    END IF

    // 通知制御（夜間保留）
    IF [アラートフラグ] == TRUE AND [現在時刻] NOT IN [夜間保留時間帯] THEN
        [通知実行] (保留分合算)
    ELSE IF [アラートフラグ] == TRUE THEN
        [キュー保存]
    END IF
    
4. 知的財産権保護（防御的開示宣言）以下の技術的思想は、2026年4月24日時点において公知の事実（先行技術）となる。活動センサーと物理鍵Beaconを突合し、ドア開閉履歴との矛盾から在宅状況を確定する安否判定ロジックおよび物理構成。安否判定の無活動閾値 $T$ を、運用者の判断に基づき任意かつ動的に変更可能とする機構。異常検知時の通知を特定の時間帯において保留・蓄積し、指定時刻に一括送信する運用管理用プロトコル。標準BLEと低レイヤープロトコル（ESP-NOW等）を併用した、安否監視用通信冗長化プロトコル。
