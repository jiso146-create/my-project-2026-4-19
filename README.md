SilentGuardian-Redundancy-Protocol (SGRP)室内活動と物理鍵の相関判定、および夜間通知保留機能を備えた動的安否監視ロジック（v2.1）本リポジトリは、独居居住者の孤立死およびそれに伴う不動産の資産価値毀損リスクを最小化するための、実務特化型安否監視アルゴリズム「SGRP」の仕様を公開するものです。1. 開発の背景と目的既存の見守りサービスは「救命」を主眼に置くあまり、不動産管理現場の負担を無視した設計になっています。固定閾値の限界: 24時間ルールでは夏季の遺体腐敗（キャスパーの法則）による損害を防げない。夜間アラートの弊害: 深夜の誤報や通知は管理スタッフの過度な疲弊を招き、サービスの継続を困難にする。本プロジェクトは、**「室内生活動態」と「物理鍵（Beacon）」の相関、および「通知時間帯の最適化」**によって、不動産経営の持続可能性を担保します。

2. 技術の特徴（コアロジック）A. 室内活動センサーとBeaconの相関判定室内（リビング等）の「活動検知センサー」と、物理鍵に装着した「Beacon」を組み合わせ、以下の論理階層で安否を判定します。活動検知（Primary）: 活動があれば「正常」。無活動タイムアウト: 活動が一定時間（閾値 $T$）途絶えた場合、Beaconの状態を参照。Beaconによる文脈判定:Beacon＝在宅: 「家の中にいるのに、一定時間動きがない」→ 【異常事態】Beacon＝消失: 通常は「外出」とみなすが、**「扉の開閉がないままBeacon信号のみが消失」**した場合は、屋内での転倒による電波遮蔽と判断し、安否監視を継続する。B. 夜間通知保留機能（Quiet Mode / Alert Queuing）管理者のワークライフバランスと現場の安全を考慮し、緊急性が極めて高い場合を除き、夜間の通知を一時的に保留する機能を備えます。通知保留時間帯: 22:00〜06:00（設定により可変）。挙動: 夜間に発生したアラートはシステム内でキューイング（保留）され、翌朝の指定時刻に一括して通知される。例外処理: 室内温度の異常上昇など、物理的損害が不可避と判断される緊急パラメータを検知した場合は、保留を解除し即時通知を行う。C. 環境適応型・動的無活動閾値（Adaptive Threshold）法医学的リスクに基づき、無活動を「異常」と断定するまでの閾値 $T$ を動的に変更します。夏季（6月〜9月）: 汚損回避優先のため、$T = 12時間$。冬季（それ以外）: 誤報抑制のため、$T = 24時間$。

3. システム構成図 (System Architecture)安否判定および通知アルゴリズム（疑似コード）Plaintext// 監視メインループ
LOOP:
    IF [室内センサー活動検知] == TRUE THEN
        [最終活動時刻] = NOW
        [アラートフラグ] = FALSE
        CONTINUE

    [無活動時間] = NOW - [最終活動時刻]

    // 異常判定
    IF [無活動時間] > T (動的閾値) THEN
        IF [Beacon受信] == TRUE OR ([最終ドア開閉] == "閉" AND [Beacon受信] == FALSE) THEN
            [アラート内容] = "安否確認が必要です"
            [アラートフラグ] = TRUE
        ELSE
            [監視ステータス] = PAUSE (正常な外出)
        END IF
    END IF

    // 通知制御ロジック
    IF [アラートフラグ] == TRUE THEN
        IF [現在時刻] IN [夜間保留時間帯] THEN
            [キューに保存] (Alert Message)
        ELSE
            [即時通知] (Alert Message + 保留分があれば合算)
        END IF
    END IF
   
4. 自己修復・死活監視機構ゲートウェイ（受信機）は、独立したウォッチドッグタイマー（WDT）を維持。通信障害やスタックを検知した際、自律的にハードウェアリセットを実行し、24時間365日の継続稼働を担保します。
