SilentGuardian-Redundancy-Protocol (SGRP)複合センサーとBeaconの冗長通信による動的安否監視ロジック本リポジトリは、独居居住者の孤立死および、それに伴う不動産の事故物件化リスクを低減するための監視アルゴリズムを公開するものです。

1. 開発の背景と目的既存の安否監視サービスには、以下の実務的な欠陥が存在します。
2. 固定閾値の問題: 夏季の高温下では、一般的な「24時間ルール」では遺体腐敗（キャスパーの法則）による汚損損害を防げない。
3. 誤報による現場の疲弊: 在宅・外出の判定が多くの場合、自己申請のため、空振り訪問が多発している。
4. 操作の強制: 高齢者にボタン操作やスマホ携行を強いるため、認知機能低下時に機能しない。

本プロジェクトは、これらの課題を「物理的な鍵への執着」という身体的習慣と「可変ロジック」で解決します。

2. 技術的特徴 (Core Logic)※本公開は技術的思想の公知化を目的としており、特定のソースコードに依存しない汎用的なロジックを定義します。
 A. 在不在の確定判定（物理鍵連動）居住者が外出時に必ず携行する「物理的な鍵」にBeacon送信機を装着。住戸内のゲートウェイがこの識別信号をスキャンし、信号の有無によって「在宅（監視ON）」「外出（監視PAUSE）」を1か0かで確定判定します。これにより、外出中の無活動による誤報を物理的に排除します。

 B. 環境連動型・可変判定閾値（Adaptive Threshold）異常と判定するまでの「無活動許容時間（閾値）」を固定せず、環境パラメータに基づいて動的に調節可能な構成とします。可変性: 運用者は、季節、外気温、居住者のリスク特性に応じて、判定閾値 $T$ を任意に変更できます。実務例: 夏季などの高リスク期には損害回避を優先して $T$ を短時間（例：12時間）に設定し、冬季や低リスク期には誤報抑制を優先して $T$ を長時間（例：24時間）に拡張する調節機構を有します。

 C. 二重プロトコルによる通信冗長化RC造マンション等の遮蔽環境における信号ロストを防ぐため、環境に応じて以下の二重化プロトコルを採用します。
Primary: 標準的なBluetooth Low Energy (BLE) 通信。Secondary: OSのオーバーヘッドを介さない低レイヤー通信（ESP-NOW等）を物理層で並行運用。これらをゲートウェイ側で突合し、一方が遮蔽されても「在宅フラグ」を維持する冗長化アルゴリズムを適用します。

3. システム構成図 (System Architecture)Input: Beacon信号 (Presence/Absence) + 各種センサー (Activity)Logic:信号あり ＋ 無活動 $ > T$ (可変閾値) → 異常発報信号なし → 監視一時停止Output: 管理者、地域包括支援センター等へのプッシュ通知
IF [Beacon受信] == FALSE THEN [監視ステータス] = PAUSE (判定タイマー停止)
IF [Beacon受信] == TRUE AND [センサー活動] == NONE THEN [タイマー] カウント開始
IF [タイマー] > [可変閾値 $T$] THEN [アラート通知] 発報
IF [現在の月] IN [6, 7, 8, 9] THEN $T$ = 12h (初期値)ELSE $T$ = 24h (初期値)※ $T$ は運用管理者の設定により任意に変更可能とする。

受信機の自己修復・死活監視 (Self-Healing Mechanism)
ゲートウェイ（受信機）は、内部に独立したウォッチドッグタイマー(WDT)およびネットワーク接続監視スレッドを保持する。Wi-Fiスタックのハングアップやメモリリークを検知した際、自律的にハードウェアリセットを実行し、24時間365日の継続稼働を担保する。また、致命的なハングアップ時にはクラウド側からのポーリング欠如（Heartbeatロスト）により、システム側から管理者へ「通信機異常」を即時通知する。
